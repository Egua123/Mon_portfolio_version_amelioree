Analyse des données BIXI Montréal et impact de la météo sur les déplacements

Résumé : Projet personnel visant à analyser les données ouvertes de BIXI Montréal pour comprendre comment la météo influence les habitudes de déplacement. Le tableau de bord Power BI combine trajets + météo pour révéler des tendances d’utilisation.

Objectifs principaux :

  • Nettoyer et modéliser plus de 2 Go de données de trajets BIXI 2025 + intégration météo (température, précipitations, vent) via Meteostat.
  • Créer un modèle de données optimisé (saison, jour, tranche horaire, stations).
  • Produire des visualisations dynamiques : stations populaires, heures de pointe, durée moyenne, corrélations météo/usage.
  • Mettre en place des KPI pour suivre fréquentation et variations selon les conditions climatiques.

Technologies utilisées : Power BI, Power Query (M), DAX, Python, CSV, Meteostat.

Compétences démontrées : ETL, modélisation, KPI, storytelling data, dashboarding Power BI, intégration de sources externes.

Système de recherche d’images basé sur le contenu (CBIR)

Résumé : Projet académique visant à concevoir un système CBIR capable de retrouver des images similaires selon leur contenu visuel. Des CNN préentraînés extraient des vecteurs de caractéristiques utilisés pour la recherche et le clustering.

Objectifs principaux :

  • Utiliser des CNN préentraînés (VGG16, ResNet50, InceptionV3) pour extraire des descripteurs robustes.
  • Appliquer un K-Means sur les vecteurs de caractéristiques pour regrouper des images similaires.
  • Développer une interface Streamlit : upload d’image + choix de distance (Euclidienne, Manhattan, Chebyshev, Canberra) + affichage des résultats.
  • Optimiser la phase offline (extraction, clustering, sauvegarde) et la phase online (requête + ranking + affichage).

Technologies utilisées : Python, TensorFlow/Keras, scikit-learn, NumPy, Pandas, Matplotlib, Streamlit.

Compétences démontrées : Deep learning appliqué, extraction de features, clustering, recherche de similarité, UI data/ML.

Système de gestion de tickets — React / Node / MongoDB

Résumé : Application web de gestion de tickets avec authentification sécurisée, rôles et endpoints REST. Le but : créer, suivre et gérer des tickets (support/incident) via une architecture claire frontend + backend.

Points clés :

  • Authentification sécurisée avec JWT et hachage des mots de passe via bcrypt.

  • Base de données MongoDB via Mongoose (modèles + validation).
  • Gestion des rôles (ex. admin / utilisateur) et permissions côté backend.

Technologies utilisées : React, Node.js, Express, MongoDB, Mongoose, JWT, bcrypt.

Compétences démontrées : Full-stack JS, sécurité auth, conception d’API, modélisation NoSQL.
